## Additive YAPK Schema Merge

- Uploaded schema was merged additively, not used as a blind replacement.
- `AppPackageInfo.required` includes `Groups`.
- `AppPackageInfo.properties.Groups` exists and `Groups[]` references `#/$defs/ListGroupInfo`.
- `ListGroupInfo` includes `ID`, `Code`, `Name`, and `Description`.
- `ListGroupInfo.required = ["ID", "Code", "Name"]`.
- `AppPortalInfo.properties.Groups` reuses `ListGroupInfo`.
- `MetadataCategoryInfo.required = ["CategoryID", "Name", "Code", "Status"]`.
- `MetadataInfo.required = ["ID", "ParentID", "Name", "Code", "Status"]`.
- Current-only hardening such as `x-yeeflow-standard-additions` was preserved.
- The uploaded `FormReports` required-list conflict was resolved by preserving the v0.6.9 product rule: `FormReports` is optional legacy storage; `FormNewReports` is current and required for generated workflow reports.
- AppID storage-code validation is enforced for generated-final YAPK checks: AppID `30` uses `setting_c`, AppID `41` uses `flowcraft`, and `TableCode`/`IndexCode` must match.

## Layout/Workspace Learning

- Three-column dashboard golden-reference learning is included.
- Three-column runtime CSS/property relearning is included.
- Style 1 dashboards keep the standard `Main > Content` rule.
- Style 2 workspace dashboards use a root `three_column_workspace_shell` rule.
- Root shell rules preserve explicit row direction, direct three-panel hierarchy, full-height bounded behavior, fixed/fill/fixed panel widths, hidden outer overflow, and scrollable panel bodies.
- Service Desk Pro real-case multi-column form workspace learning is included.
- The deep form workspace property map is included for fixed navigation, selected-record/detail panels, action surfaces, per-column scroll, and overflow behavior.

## YAP Form-Workspace Generation Hardening

- Type `105` navigation must materialize approval/form workspace pages.
- `Data.Forms[].ListID` remains `0` for generated materialization-safe workspace forms.
- Form keys, process keys, `defkey`, and navigation targets must align.
- Use `formdef.children`, not unsupported `formdef.controls`.
- Workflow graphs use a minimal valid `Start -> SequenceFlow -> End` path when a workflow shell is needed.
- Form workspace pages hide the header, use full width, and use zero page padding.
- Inline Icon/Text/Button/Dynamic field controls must use inline sizing in dense workspace rows.
- Each workspace column owns its own scroll/overflow behavior; shell overflow is bounded.
- Designer-safe control families are required; unproven control names fail generated-final checks.
- `ListDatas` must use record-ID-keyed records, not a `Datas` wrapper.
- Sample row keys must match defined field names.
- Field `Rules` remain stringified JSON.
- Service Desk/v8-safe field numbering is preserved to avoid async materialization drift.
- Native `Title` field conflicts remain warning-only where the YAP import path allows native Title preservation checks without blocking accepted materialization patterns.

## ReplaceIds Final Coverage

- YAP `Resource.ReplaceIds` must be rebuilt from the final decoded package after every generation, ID remap, repair, or baseline copy step.
- Generated-final ReplaceIds coverage must include child list IDs, layout IDs, field IDs, sample record IDs, and form/process/workflow generated IDs where applicable.
- Generators must not copy `ReplaceIds` from a baseline package after changing the decoded package.
- Equivalent YAPK final decoded `AppPackageInfo` ID/remap coverage must be validated before signing, install, import, or upgrade work. API acceptance is not runtime proof.

## Proof Boundary

This integration is repository/package validation only. No live Yeeflow import, install, upgrade, or write API call was run. Final tag creation and `stable` promotion remain pending approval.
