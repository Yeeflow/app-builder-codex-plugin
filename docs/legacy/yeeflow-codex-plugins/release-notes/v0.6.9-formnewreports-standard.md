# Yeeflow Builder Plugin v0.6.9

v0.6.9 applies the product-team clarification that `FormReports` is legacy workflow report storage and `FormNewReports` is the current workflow report collection.

## Schema Decision

- `FormNewReports` remains required in `AppPackageInfo`.
- `FormReports` is no longer required for generated YAPK apps.
- `FormReports` remains an optional legacy property for backward compatibility when old packages include it.
- Generated packages should not be forced to include empty `FormReports`.
- Generated workflow report resources must be written to `FormNewReports`.

## Validation Updates

- Generated-final validation passes when `FormNewReports` is present and `FormReports` is absent.
- Generated-final validation fails when `FormNewReports` is missing.
- Packages with workflow report content only in legacy `FormReports` fail because current workflow reports must use `FormNewReports`.
- Existing v0.6.8 YAPK schema v5 hard gates remain enforced for approval `DefResource` encoding, deployed approval IDs, page hydration, and workflow graph hydration.
- Existing v0.6.7 workflow task URL page-type gates and v0.6.6 choice-option runtime-shape gates remain enforced.

## Proof Boundary

This release was validated locally with schema, validator, source/dist, fixture, archive, and safety checks. No live install, upgrade, or runtime render proof was run during the release flow.
