# Yeeflow Builder Plugin v0.6.8

v0.6.8 updates the active canonical YAPK schema to v5 and treats the schema's `x-yeeflow-standard-additions` block as enforceable generation and validation standards.

## What Changed

- `schemas/yapk-schema.json` and the bundled dist mirror now contain v5 YAPK schema content.
- The active schema keeps the canonical filename; no active code path references `yapk-schema_v5.json`.
- The v5 FormReports overlay is materialized because `AppPackageInfo` requires `FormReports`.
- Generated approval YAPK packages now hard-fail if approval `DefResource` is not `base64("::brotli::" + Brotli(JSON.stringify(ProcessDefResourceInfo)))`.
- Deployed approval forms now require non-zero API-issued `DefResourceID` and `DeployedDefID`.
- Decoded approval `pageurls[].formdef.id` must match `pageurls[].id`.
- Decoded approval workflow graphs must include a Start node, sequence connectors reachable from Start, and workflow nodes that can hydrate in designer.
- The v0.6.7 workflow task URL page-type gates and v0.6.6 choice-option runtime-shape gates remain in place.

## Proof Boundary

This release was validated locally with schema, validator, source/dist, fixture, archive, and safety checks. No live install, upgrade, or runtime render proof was run during the release flow.
