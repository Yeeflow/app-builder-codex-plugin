# Yeeflow Builder Plugin v0.6.6 Choice Option Runtime Fix

v0.6.6 is a final patch release for the official Yeeflow Builder plugin.

## Root Cause

The v0.6.5 Data List Generation Standard required populated options for choice-like fields, but generation and validation accepted options written only to `Rules.Options`. Yeeflow runtime did not use that path for the affected single-select and multi-select fields, so the generated fields displayed blank option lists even though local structural validation passed.

## Fix

- Choice-like generated fields now use runtime-visible `Rules.choices`.
- Validators treat missing runtime-visible choices as blocking.
- Legacy `Rules.Options` alone is no longer enough for generated-final validation.
- Sample values must resolve to the configured runtime-visible option set.
- Account Health smoke fixtures were updated to exercise the corrected shape.

## Release Boundary

This release refreshed the official plugin archive and validated the corrected package locally, including signing and verifysign. No live install or upgrade API call was run in the v0.6.6 release flow.

The previous final tag `yeeflow-builder-plugin-v0.6.5` remains unchanged.
