# Phase 5Y Data List Scalar Resource Identity Shadow

## Boundary

The internal Core shadow accepts a scalar projection, resource scope, and field ordinal. It returns a frozen pre-ID field-record fragment and deterministic FieldID request. The internal Local Runtime shadow validates host-supplied lossless string allocations and returns a fresh field record.

## Evidence

Five projected Text, Datetime, Decimal, Bit, and choice branches match the retained Legacy scalar record lowerer. Nineteen-digit ListID and FieldID values remain strings. Barcode errors and lookup, sublist, identity, file, and image exclusions remain outside this shadow.

## Non-Goals

No production route, public export, distribution artifact, adapter, lookup resolution, ID allocation, template mutation, package writing, or generated-resource mutation changed.

## Phase 5AA Follow-Up

The shadow now has an accepted audit-only routing boundary. The future bridge is limited to the existing scalar branch between the Core scalar projection result and the retained Legacy field-record lowerer. This report remains proof of shadow parity only; it does not authorize production routing.
