The authoritative Create Balance export proves one exact Legacy-compatible static vertical: the embedded `user` column `field_7` lowers to an `identity-picker` with only list-field scope metadata. Phase 16 routes only this declarative metadata through frozen Core intent and fresh Local Runtime lowering. Identity resolution, selection, user/people lookup, events, validation timing, template/resource mutation, and package output remain host or product-runtime owned; no runtime parity is claimed.

The same export proves file, organization, location, and image shapes, but they are not selected: file has export-only `ver: 1` absent from current Legacy lowering; the current Legacy Sublist normalizer does not emit organization-picker, location-picker, or icon-upload. Barcode and nested Sublist are product-unsupported for this surface and have been marked as exclusions, not future candidates.
